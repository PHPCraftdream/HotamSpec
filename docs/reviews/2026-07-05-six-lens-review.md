# Шестилинзовое ревью Hotam-Spec — 2026-07-05

Шесть независимых fxx-агентов (онтология / enforcement / оператор-UX / бизнес-дверь / инженерия / vision),
~1.07M токенов исследования, read-only. Синтез координатора. HEAD на момент ревью: 1b710f3 (219 SETTLED, 195 ENFORCED).

## ГЛАВНОЕ ОТКРЫТИЕ (сошлись две линзы независимо)

**tmp/mm — это настоящий бизнес-кейс, и пора применить методологию по назначению.**

- Бизнес-линза: `tmp/mm/` = два профессиональных спека (~109K) реального проекта **AICAD** —
  246 контрактных требований → 580 release-stage срезов, UAT/ПСИ, 6 ворот, 7 агентских ролей.
  Идеологическое совпадение поразительное: таблица режимов агента = дословно R-ai-presents-not-decides;
  Acceptance Decision Log = Conflict.DECIDED+decided_by; материал кишит осями
  (contract-fidelity-vs-product-structure, uat-vs-formal-psi, agent-autonomy-vs-approval).
  **Правильный пилот: домен `aicad-governance` ~30-50 атомов** (Jira остаётся системой фактов,
  Hotam — governance-слоем напряжений/решений/допущений). НЕ тащить 826 инстансов в граф.
- Vision-линза (финал, «одна вещь на 30 дней»): завести ОДИН чужой настоящий бизнес-домен
  (в идеале со вторым живым человеком-стьюардом), довести до первых десяти конфликтов, не трогая фреймворк.
  «Система на 219 атомах самоописания превосходно инструментирована и ни разу не была применена
  по заявленному назначению; каждый месяц самошлифовки полирует зеркало, а не проверяет,
  методология это или автопортрет».

## СКВОЗНЫЕ ТЕМЫ (ранжированы по числу сошедшихся линз × тяжести)

### T1. Подпись выродилась в бланк (4 линзы: vision, enforcement, онтология, бизнес)

- DEL-1 «реши все задачи…until resolver revokes» — семантически стоячая предавторизация «на всё»;
  C-be22cdd1 DECIDED со ссылкой на неё = ИИ решил конфликт по бланку. Инварианты проверяют ПОЛЕ, не АКТ.
- `decided_by` — свободная строка; git-authorship не проверяет ничто; агент может вписать любую подпись.
- Подпись AssumptionTransition НЕ доживает до субстрата: decided_by остаётся в gitignored JSON
  (writer пишет только status+reason). R-trust-anchor-mechanism [E] неаудитируем из графа.
- HELD→DECIDED **стирает Variant-ы** (C-be22cdd1: variants=() после решения; implies/costs
  не-выбранных альтернатив уничтожены — анти-релитигационный груз потерян).
- Во всей онтологии НЕТ ни одного timestamp. Даты решений — в прозе. UNCERTAIN-aging меряет
  fan-out, не возраст.
- Нет surface активных делегаций и их возраста в what_now.
- Вектора: **Decision/Signoff frozen-рекорд** {decided_by, date, verbatim, instrument: personal|DEL-n,
  chosen_variant?} прикрепляемый везде (Conflict DECIDED/HELD, AssumptionTransition, Delegation);
  привязка к git-authorship; шов под криптоподпись; SignatureBudget (по образу ContextBudget) —
  бюджет подписей стьюарда как моделируемый ресурс.

### T2. Система закрывает быстрее, чем находит (vision, UX, онтология)

- «graph clean» как добродетель = анестезия: 0 OPEN, 0 UNCERTAIN (два крупнейших сомнения
  перетипованы в IMPLEMENTS — статус, который по конструкции НЕ стареет, не даёт fallout,
  не имеет меры прогресса = легальный тихий угол).
- Машинный поиск напряжений: за всю историю лексический аудит нашёл 1 из 9 конфликтов.
  Плотность конфликтов 3.7% — подозрительно мало для живой системы.
- attention-канал ЛЖЁТ: 3 из 5 «awaiting resolver» уже посажены и закоммичены (wave16-*.json
  не заархивированы после batch-каруселей); тикеты T-1/T-2 висят на сделанные волны.
- Вектора: **LLM-в-петле как генератор подозрений** (семантический свип пар атомов →
  ProposedConflict-подозрения в pending/, квота ≤3/волну; шов R-attention-superset-of-diagnose
  УЖЕ построен для этого); **время в узлах** (settled_at/decided_at + обязательный распад
  IMPLEMENTS без прогресса за N волн + «давность последнего confront» для SETTLED);
  архивация лендженных pending (проверять target_anchor против графа).

### T3. Операционная реальность невидима графу — самоизмена генеративному закону (vision, UX, бизнес)

- Перф-кампания (3 коммита, T2 223s→56s) — крупнейшая работа суток — НЕ имеет ни узла, ни тикета;
  при этом она материально сняла напряжение ЕДИНСТВЕННОГО открытого конфликта C-ec1ec532
  (speed-vs-verification) — конфликт держится DETECTED в графе, а решён в коде мимо графа.
- spawn-log.jsonl физически отсутствует при ~30+ реальных спавнах (агенты этой сессии тоже неучтённые).
- Два трекера работ, оба полумертвы: тикеты созданы 03.07 и НИ РАЗУ не двигались; живая работа —
  в Claude-TaskList. R-user-request-decomposed-to-tickets исполняется декларативно.
- Метры без последствий: boot-cite 10-13/128 (90% cited:false) — читатель не подключён НИЧЕМ;
  context UNMEASURED днями; паттерн «построить измеритель → не смотреть».
- hotam-dev застой (9 атомов) = канарейка adoption: внимание живёт только в активном домене.
- Вектора: **мост логи→граф** (land/spawn/ticket-логи автоматически предлагают wave-сущности
  и метрики-допущения); boot-cite rate → P0-Finding в reflection; кросс-доменный what_now.

### T4. Enforcement-периметр не замкнут — чекер судит себя (enforcement, инженерия)

- **Выпотрошенный enforcer невидим**: `return []` в любом не-core check_* проходит ВЕСЬ мета-слой
  (check_method_matches_docstring пропускает тела без Violation-сайтов; биекция довольна именем).
  Fires-тестов 51 на ~90 чеков.
- **Гард закрывает только graph.py**: invariants.py, gate.py, enforcer_resolution.py, тесты,
  baselines, .active-domain — редактируются свободно. «Красное чинится ослаблением чекера».
- enforced_by резолвится ГРЕПОМ ТЕКСТА (имя в комментарии = «резолвится»; тест с телом pass = ENFORCED).
- closure опциональна (--triggering-kind передан в 5% лендов); R-verify-closure-per-action [E] на бумаге.
- land-log в gitignored .runtime/: пустой лог → gate_status «SATISFIED» (fail-open!); на свежем
  клоне гарантия испаряется.
- Ratchet-протечки: компаундность лексическая, baseline дописываем, тест редактируем.
- 33% лендов красные (ре-драфт карусель); auto-revert «not implemented» — красный ленд
  оставляет graph.py записанным.
- Вектора: **замкнуть периметр самомодификации** (гард/хэш-пин на invariants.py+gate.py+tests/**+baselines);
  «ENFORCED = доказуемо кусается» (RULE без Violation-сайтов = Violation; биекция check↔fires-тест;
  AST-резолв вместо грепа); closure по умолчанию; gate_status fail-closed на пустом логе;
  транзакционный LAND (all-or-nothing + revert).

### T5. Кристалл раздут, сенсориум шумит и дублирует (UX, инженерия)

- Кристалл 75.7k: рабочее ядро каждого хода ~9k; 75% — справочники O(атомов), дублирующие docs/gen;
  THINKING-дистилляты обрезаны мид-словом («filters/predicat…») — иллюзия знания;
  рост +7.8k/день → **warn 130k через ~7 дней**. Закон «дерева ссылок» уже DECIDED — пора исполнять.
  Оценка среза: **55-60% без потери силы** (75.7k → ~30-35k).
- **БАГ: двойная проводка хуков** — committed settings.json + settings.local.json исполняются ОБА:
  emit_cipher дважды на промпт, gen_spec дважды на SessionStart, boot-cite-log полностью задвоен,
  плюс локальный PostToolUse context_producer через uv (~2с) на КАЖДЫЙ tool call.
- Четыре канала одного пульса (LIVE-STATE резидентный + emit_cipher + diff_watch + what_now).
- P6-хвост повторяется дословно каждый промпт, дедупа нет; порог revisit «+10 SETTLED» при темпе
  8/день перевзводится каждые 1.5 дня.
- confront.py слеп к русскому (токенизатор [a-z0-9]; русская формулировка против R-budget-measure →
  «likely novel, proceed»); полуслеп к парафразу (0.1611 при пороге 0.15).
- Петля 6 шагов театральна для read-only входов; узаконить 2 режима (full для мутаций, short-circuit для чтения).
- Вектора: **дельта-сенсориум** (инжект только нового/изменившегося против снапшота, стабильное — счётчик);
  **кристалл = семя+указатели** (реализация «дерева ссылок»); вычистить пересечение local/committed хуков;
  единственный пушер пульса; confront: не-ASCII>30% → «lexical check inapplicable».

### T6. Линия жизни знаний не типизирована (онтология, enforcement)

- **Анти-релитигация стоит на em-dash-регэкспе**: «REJECTED — REPLACES» матчится точной строкой;
  6 из 32 REJECTED с двойным дефисом «--» МОЛЧА ВЫПАЛИ из RECENTLY-REJECTED кристалла.
  Ребра `replaces` нет в RELATION_KINDS.
- Половина коллекций графа без механического write-path (Operator/Goal/Process/EntityInstance
  create; Conflict.members update) — wave-инстансы hotam-dev доказуемо вручную вброшены;
  «Conflict переживает churn членов» не имеет механизма; у DECIDED C-c3911f28 ОБА member — REJECTED-надгробия,
  ничто не подсвечивает.
- Assumption обходит keystone §Lifecycle (плоский frozenset + подписной замок прозой в докстринге).
- Payload-в-строке: OPEN(question)/DECIDED(rationale)/HELD(reason)/REVISIT_WHEN(cond) — парсинг
  strip("()") ломается на скобках; prefix_states-хак существует только ради этого.
- Identity: Requirement-слаг без graph-wide уникальности; реестра анкор-префиксов нет
  (V4/V5-вердикты сквотят V-неймспейс Variant; DEL- сквотнут тикетом).
- Вектора: **replaces-ребро** + генерация RECENTLY-REJECTED/confront из рёбер; **NODE_SCHEMAS-реестр**
  (kind → prefix, коллекция, ref-поля, lifecycle) с генерацией typed-anchor/dangling/lifecycle-чеков
  и скелетов Proposed* — режет налог на новый тип на порядок; сохранение Variant при DECIDED
  (+chosen_variant поле); ASSUMPTION_LIFECYCLE как Lifecycle с requires_signoff на Transition.

### T7. Бизнес-дверь заперта конкретными замками (бизнес)

- Enforcement привязан к pytest ЭТОГО репо → бизнес-атомы навечно PROSE, burn-down мёртв для бизнеса.
  Вектор: **протокол Enforcer-адаптеров** (JQL/SQL/API-проба/ручная аттестация с истечением).
- Нет data plane: импорт 246 требований = 826 AST-вставок в Python-файл. Вектор: import_entities
  (CSV→батч, один гейт на батч), инстансы как данные отдельно от graph.py.
- Онтологические дыры: ENTITY_FIELD_KINDS без date/money/duration; нет количественных целей (SLA).
- Onboarding реального домена = 16-28 часов (не «15 минут»); нужен ассистент «проза→атомы»
  (LLM-прескрининг + confront + подпись) и демо-домен-витрина.
- Честная ниша (когда tension-graph незаменим): (1) долгоживущие противоречивые обязательства
  с формальной приёмкой; (2) AI-оператор ежедневно в репо, конституция машинно; (3) решения
  перепринимаются, допущения гниют. Иначе ADR+трекер дешевле — говорить честно.

### T8. Инженерные долги (инженерия; UX подтверждает LAND-боль)

- Тройная реализация резолва активного домена (graph.py/gen_spec/apply_proposal) — синхронизация руками.
- Библиотека hotam_spec ходит по ФС репозитория: all_violations = O(репозиторий), на каждый промпт
  в холодном интерпретаторе. При ×3 хуки уйдут в многосекундные задержки.
- graph.py домена — 4097 строк (19 строк/атом) под построчно-колоночной AST-хирургией (~1500 строк
  в apply_proposal); магнит merge-конфликтов; при ×3 → 12k строк.
- Каждый LAND: gen_spec перегенерирует ВСЕ домены; T2 принудителен для каждого нового узла
  (T1-выигрыш испаряется ровно в фазе роста).
- gen_spec 4852 строки / 9 ответственностей + ~сотни строк МЁРТВОГО кода (8 легаси-апдейтеров с 0 call-sites).
- rules-as-data — аннотация, не движок: 38 «табличных» чеков остаются рукописными клонами
  (~900-1000 строк дублей); блокер — Jaccard-мета-линт 0.05 (почти нулевая различающая сила).
- Тесты: ~25-30% балласта по числу (~10% по времени); 194 collected-элемента одного линт-теста;
  30 файлов / 76 call-sites грузят граф заново (exec 4097 строк) вместо session-фикстур.
- Вектора: единая инфраструктурная плоскость (domain_resolution/repo_paths/claude_md в библиотеку,
  tools как пакет); распил gen_spec (в т.ч. немедленное удаление мёртвых апдейтеров) и invariants
  (табличный исполнитель); отвязка стоимости от масштаба (mtime-индекс enforcer→node-id,
  компилированный снапшот графа по hash, dirty-домены, семейный гейт для new-node).
